## v1.4.0-dev
- 与正式版 v1.4.0 同步更新

## v1.4.0
### 新增功能
- 自动检测更新（定时检查正式版更新）
- 一键安装更新（下载 → 替换 → 重启）
- 开发版更新通道（单独按钮、单独下载链接）
- 版本日志面板（正式版/开发版切换查看）
- 系统信息面板（显示版本、PID、数据目录）

### Bug 修复
- 定时同步内存泄漏（每 10 分钟心跳检测）
- 配置保存失败（跨线程访问冲突）
- 启动时不携带 headers 导致 401

### 优化
- 前后端分离：前端从 `/app` 加载，后端从 `/api` 调用，源码结构更清晰
- 定时同步不再受长期运行拖累
- 版本号读取统一为 `main.py` 中的 `VERSION` 常量
- GitHub Actions 自动打包（开发版/正式版双通道）
- 自定义导出目标目录

## v1.3.0
### 新增功能
- 新的 QCE 导出：适配 `POST /api/messages/export`
- QCE 版本检测（旧版 v3 → v4 两套 API 兼容）
- 多时段定时策略：支持工作时段/晚间/周末不同步频

### Bug 修复
- 导出任务 ID 接口路径变更（`/api/tasks/{id}` → `/api/tasks`）
- 任务状态异常时显示详细错误
- 适配不含 `statistics` 字段的导出 JSON

### 优化
- `timeRangeDays` 支持同时处理旧格式和时间戳格式
- 导出文件路径支持时间模板（`{peer}`, `{uid}`, `{date}`, `{time}`）
